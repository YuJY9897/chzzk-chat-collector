# Changelog

이 프로젝트의 주요 변경사항을 기록합니다.

## 2026-07-12 (6차) — Electron 데스크톱 앱 (v0.2.0)

- 브라우저 UI를 대체하는 네이티브 데스크톱 앱 추가 (`npm run app`, exe 빌드는 `npm run dist`)
- 기존 기능 전부 이식: 계정 연결, 로그 수집 ON/일시정지/재개/종료, 방송 종료 자동 감지, 저장 경로 선택(네이티브 폴더 창)
- 창 닫기 시 "백그라운드로 유지 / 완전히 종료 / 취소" 선택 대화상자 + "이 선택 기억하기" (트레이 메뉴에서 초기화 가능)
- 트레이 아이콘 내장 (상태 표시, 더블클릭 열기, 앱 종료), 수집 종료 시 Windows 알림
- 실시간 채팅 목록 (IPC 푸시 방식이라 새로고침 불필요), 마지막 수신 상대 시각 표시
- 포터블 exe 패키징 (dist/CHZZK-Clip-Scout.exe, .env를 exe 옆에 두고 실행)
- 웹 버전(npm run server)은 레거시로 유지

## 2026-07-12 (4차) — 트레이 아이콘

- 백그라운드 실행 중임을 보여주는 트레이 아이콘 추가 (초록 점, 작업표시줄 오른쪽 아래)
- 더블클릭 → 화면 열기, 우클릭 메뉴 → 열기 / 앱 종료
- 서버가 종료되면 트레이 아이콘도 자동으로 사라짐, 중복 실행 방지
- 추가 설치 불필요 (Windows 내장 PowerShell + .NET Forms 사용)

## 2026-07-12 (3차) — 사용자 관점 다듬기

- UI 전면 리디자인: 카드형 레이아웃, 상태 배지(수집 중 펄스 표시), 다듬어진 버튼/입력 스타일
- 수집 종료 알림을 브라우저 기본 alert 대신 앱 내 모달로 교체 (저장 위치 표시 + 저장 폴더 열기 버튼)
- "저장 폴더 열기" 기능 추가 — 수집 결과 파일을 탐색기에서 바로 확인
- 저장 위치 기본값을 절대경로로 표시해 어디에 저장되는지 명확히
- 최근 채팅/수집 시각을 한국 시간(HH:MM:SS)으로 표시, 마지막 수신은 "N초 전" 상대 시간
- "다시보기 기준 시작 시간" 입력을 고급 설정으로 접어 초심자 혼란 감소
- 계정 미연결 시 로그 수집 ON 버튼 비활성화
- 중복 실행 파일(숨김 실행.vbs) 제거 — cmd 하나로 통일 (자체적으로 백그라운드 실행)
- 전체 플로우 실계정 테스트: 연결 → ON(구독 성공) → 일시정지 → 재개 → 종료 → 폴더 열기 → 앱 종료

## 2026-07-12 (2차) — UI 개편 + 치명적 버그 수정

### 버그 수정 (치명적)
- **치지직 세션 이벤트가 JSON 문자열로 도착하는데 객체로 가정하고 있던 버그 수정.** 이 버그 때문에 기존 버전은 sessionKey를 읽지 못해 실제 환경에서 채팅 구독이 아예 되지 않았음 (6월 테스트 CSV가 비어 있던 원인). 실제 치지직 세션으로 구독 성공 확인 완료.

### UI/UX 개편
- 실행 시 CMD 창 없이 백그라운드로 동작, 웹 화면의 "앱 종료" 버튼으로 종료
- 계정이 연결돼 있으면 연결 버튼 비활성화 + "이미 연결되어 있습니다" 안내
- "자동 저장 예약" → "로그 수집 ON/OFF"로 개념 단순화. 방송 전에 ON 하면 방송 시작 시 자동 수집
- 저장 경로 지정 기능 추가 (수집 중에는 변경 불가)
- "채팅 저장 멈추기" → "일시정지/재개" + "종료" 버튼으로 분리. 일시정지 후 재개하면 같은 파일에 이어서 저장 (JSONL에 pause/resume 마커 기록)
- 방송 종료(구독 해제) 자동 감지 → 수집 자동 종료 + 저장 위치를 알림 창으로 표시
- "최근 채팅" 새로고침 버튼 추가 — 원할 때만 수집 상태/최근 채팅 확인

## 2026-07-11

### 안정성
- 방송 중 네트워크 순단 시 자동 재연결 추가: 5초 간격, 최대 5분, 같은 파일에 이어쓰기
- 연결 시도 실패(`connect_error`/`connect_timeout`)도 재시도 루프에 포함 — 재연결이 조용히 멈추는 버그 수정
- 재연결 구간을 JSONL에 `reconnect_start`/`reconnect_end` 마커로 기록해 수집 공백 확인 가능
- `apiFetch`가 비-JSON 응답(HTML 에러 페이지 등)에도 안전하게 동작

### 개인정보 / 보안
- 시청자 식별자(`sender_channel_id`)를 SHA-256 해시(16자)로 익명화
- CSV 수식 인젝션 방지 (`=` `+` `-` `@` 시작 셀에 `'` 접두)

### 사용성
- 수집 중 웹 화면에 "마지막 채팅 수신 시각" 표시 (새로고침으로 정상 수집 확인)
- 새 수집 시작 시 이전 방송의 수신 시각/최근 채팅 초기화

### 구조
- CLI 수집기(`collector.js`)가 `ChatCollector` 클래스를 재사용하도록 정리 (중복 로직 제거)
- CLI 토큰 갱신 시 `.tokens.latest.json` 저장 복원 (콘솔 출력만으로는 토큰 유실 위험)
